# T10.1.36 – Safety & WAIT-Handling (Host/UI)

## Ziel
Einführung einer konsistenten WAIT-Logik bei Door-Events sowie saubere Trennung von RUNNING / WAIT / STOPPED
im UI und Host.

## Inhalt
- Door-Open erzwingt sofort WAIT
- Pause-Button wird door-abhängig gesperrt
- UI-Status folgt strikt dem Oven-Runtime-State
- Keine UI-eigene Aktuatorlogik

## Ergebnis
- Kein Resume bei offener Tür möglich
- Konsistentes Safety-Verhalten Host ↔ Client
- Grundlage für spätere Overtemp-Logik
